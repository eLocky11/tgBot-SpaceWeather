# 🌌 Space Weather - TG Bot

A Telegram bot for space weather monitoring. It polls the [NASA DONKI API](https://api.nasa.gov/DONKI/notifications), filters new events, and sends formatted HTML notifications to a Telegram channel in Russian.

The bot is not a long-running process - it is designed to be triggered on a schedule (cron), runs one full cycle, and exits.

## How the Bot Works

Each run follows this pipeline:

```
NASA DONKI API
      │
      ▼
  1. Fetch         - async request to DONKI via aiohttp
      │
      ▼
  2. Deduplicate   - check messageID in SQLite, skip already-sent events
      │
      ▼
  3. Format        - regex-parse messageBody, render matching Jinja2 template
      │
      ▼
  4. Send          - deliver HTML message via python-telegram-bot
      │
      ▼
  5. Persist       - store messageID in DB, prune records older than 180 days
```

### Implementation Details

- **`src/paths.py`** - the single source paths and URLs.
- **`src/donki.py`** - two methods: `fetch()` (synchronous, unused in the main loop) and `new_fetch()` (async, the active one).
- **`src/database.py`** - SQLite with WAL mode. The `events` table stores `messageID`, the rendered message body, and a timestamp. `remove_old_events()` prunes records older than 180 days on every run.
- **`src/formatter.py`** - static class `Formatter`. `extract_context()` pulls fields from the plain-text event body using regex; `get_template()` returns the appropriate Jinja2 template by event type.
- **`src/templates/templates.py`** - Russian-language templates in Telegram HTML parse mode. The `DefaultUndefined` class substitutes `"Неизвестно"` ("Unknown") for any missing template variable, preventing render errors.
- **`src/notifier.py`** - subclasses `telegram.Bot`, wraps `send_message` with logging.

## Setup & Usage

### Requirements

- Python 3.10+
- Internet access (NASA API + Telegram API)

### Installation

```bash
git clone https://github.com/eLocky11/tgBot-SpaceWeather.git
cd tgBot-SpaceWeather
pip install -r requirements.txt
```

### Configuration
Create a `.env` file in the project root:
```env
TG_BOT_SW_TOKEN=...    # Bot token from @BotFather
TG_CHAN_SW_ID=...      # Production channel/chat ID
TG_CHAN_TEST_ID=...    # Test chat ID
NASA_API=...           # NASA API key (free at api.nasa.gov)
```

Paths and URLs are defined in `src/paths.py`:
```python
DB_PATH   = "data/db.db"
DONKI_URL = "https://api.nasa.gov/DONKI/notifications"
```

### Running

```bash
# Production (writes to data/db.db, sends to TG_CHAT_ID)
python app.py

# Test mode (in-memory DB, sends to TG_TEST_ID)
python app.py test
```

Test mode does not persist any state between runs - useful for debugging templates and regex parsing.

### Scheduled Execution (cron)

```cron
# Run every 30 minutes
*/30 * * * * /usr/bin/python3 /path/to/tgBot-SpaceWeather/app.py
```

## Adding a New Event Type

Three files must be updated in sync:

1. **Template** - add `XXX_TEMPLATE = Template(...)` in `src/templates/templates.py`
2. **Parsing** - add an `elif ev["messageType"] == "XXX":` branch in `Formatter.extract_context()` in `src/formatter.py`
3. **Mapping** - add `"XXX": XXX_TEMPLATE` to the dict in `Formatter.get_template()`
